Project Requirements Document (PRD): Vestaboard Split-Flap Display Emulator App

Document Version: 1.0

Date: March 26, 2026

Author: Cozy (based on exact video review from original X post)

Project Goal: Recreate the exact software application demonstrated in the ~32-second video from the X post . The app must turn any TV into a fully functional, pixel-perfect digital replica of a physical Vestaboard split-flap display, matching every visual, animation, audio, and behavioral detail shown in the demo.

1. Video Review Summary (Exact Analysis of the Demo Video)

The 32-second MP4 video demonstrates a polished, full-screen software emulator running on a large TV in a living-room-like setting. Key observed elements:

  • Display Layout: A 6-row × 22-column grid (132 character tiles total), exactly matching real Vestaboard hardware dimensions.
  • Aesthetic: Dark retro airport-terminal look (black/near-black background, high-contrast white/cream flaps with subtle 3D depth, metallic or matte hardware-inspired borders and subtle lighting/shadows on each tile).
  • Animation Behavior: Characters transition via realistic split-flap mechanics. Each tile’s top and bottom flaps rotate independently with physical momentum (slight overshoot, bounce/settle effect). Flaps cycle sequentially through the full character set (A-Z, 0-9, punctuation, symbols, spaces) in order — not random jumps. Multiple tiles animate simultaneously but with natural staggered timing for a mechanical, organic feel. Transitions are smooth at 60 fps with no jitter.
  • Audio: Perfectly synchronized mechanical clack/click sounds (realistic split-flap hardware audio) for every flap movement. Volume and timing match the visual flips exactly; no lag or desync.
  • Content Shown: The demo cycles through dynamic messages, including an inspirational quote (“Stay Hungry. Stay Foolish. – Steve Jobs”) that assembles tile-by-tile. Implied support for weather, stats, or other feeds (text populates the full grid).
  • Interaction/Mode: Full-screen, always-on TV mode with no visible UI controls during display (configuration happens off-screen or via companion app/remote). Seamless content updates without interrupting the retro aesthetic.
  • Performance: Zero lag, instant response on modern hardware; optimized for large 4K/1080p TVs.
  • Other Details: Subtle tile shading/highlights mimic real physical lighting. No watermarks, logos, or overlays in demo mode. The entire experience feels indistinguishable from a $3,499 physical Vestaboard.

The video’s core value is the hyper-realistic emulation that makes a software solution feel like expensive hardware.

2. Business/High-Level Objectives

  • Deliver an exact 1:1 software recreation of the video demo.
  • Enable users to turn any smart TV, Apple TV, Roku, Fire TV, or cast-enabled screen into a Vestaboard-style display.
  • Support dynamic content (quotes, weather, stats) with zero subscription model (one-time purchase irrelevant for this technical PRD).
  • Prioritize fidelity: visuals, physics-based animation, and audio must be indistinguishable from the video.

3. Functional Requirements

3.1 Core Display Engine

  • Render a fixed 6×22 grid of split-flap tiles.
  • Each tile must simulate two independent rotating flaps (top and bottom halves) with:
    • Sequential character cycling (A-Z → 0-9 → symbols → space, in ASCII/Unicode order).
    • Physics-based animation: variable rotation speed, momentum overshoot, and settling bounce (exact match to video timing ~0.3–0.8 seconds per full flip).
    • 3D shading, highlights, and subtle shadows that respond to virtual lighting for depth.
  • Support full-grid messages up to 132 characters (including spaces and punctuation).
  • Background must be solid dark retro terminal color (exact hex match to video if sampled; otherwise deep black with slight vignette).

3.2 Animation & Transition Logic

  • When updating text, tiles flip sequentially per position (left-to-right, top-to-bottom with micro-stagger for realism).
  • No instant character swaps; every change must use the full mechanical flap cycle.
  • Support idle “live” mode where random non-disruptive flips can occur (matching airport-terminal vibe) or stay static until new content.
  • 60 fps minimum; buttery-smooth on 4K TVs.

3.3 Audio System

  • Embed or stream high-fidelity mechanical clack/click sound effects.
  • One unique sound per flap movement, perfectly synchronized (±10 ms) to visual animation.
  • Volume adjustable but defaults to video demo level; optional mute.
  • Loopable ambient mechanical hum optional for extra immersion.

3.4 Content Management

  • Built-in library of inspirational quotes (at minimum include the exact Steve Jobs quote from the video).
  • Support for:
    • Weather data (current conditions, temperature, location-based).
    • Stats/metrics (e.g., stock prices, crypto, sports scores, custom APIs).
    • Custom user text input.
  • Scheduling: auto-rotate between quotes, weather, stats on configurable timers (e.g., every 30–60 seconds).
  • Easy configuration via companion mobile/web dashboard or on-TV remote menu (hidden during normal display).

3.5 TV Integration & Deployment

  • Full-screen, borderless mode optimized for TVs (1080p and 4K).
  • Support for casting (AirPlay, Chromecast, Miracast) or native apps for Apple TV, Roku, Fire TV, Android TV, Samsung Tizen, LG webOS.
  • Auto-detect TV resolution and scale grid perfectly.
  • Always-on standby behavior (no sleep mode).

3.6 Configuration & Settings (Non-Display Mode)

  • Simple settings screen to:
    • Input custom messages.
    • Connect weather/API keys.
    • Adjust flip speed, sound volume, theme (dark/light variants if shown in video extensions).
    • Schedule content rotation.
  • Export/import message presets.

4. Non-Functional Requirements

  • Performance: Must run at 60 fps on mid-range TVs/ streaming devices; <50 MB memory footprint.
  • Platform: Cross-platform (priority: web-based PWA for casting + native TV apps). Primary target: Apple TV / smart TVs.
  • Accessibility: High contrast; optional high-visibility mode.
  • Reliability: 99.9% uptime in always-on mode; graceful error handling for API failures (fallback to last known content).
  • Security: Local-first where possible; secure API calls for weather/stats.
  • Audio/Video Sync: Frame-accurate synchronization (use Web Audio API or equivalent for precise timing).
  • Assets: All graphics, fonts, and sounds must be original or licensed to avoid IP issues while matching video fidelity exactly.

5. Technical Stack Recommendations (for Exact Recreation)

  • Frontend: HTML5 Canvas + CSS 3D transforms or Three.js/WebGL for realistic flap physics (or SwiftUI for Apple TV native).
  • Animation: GSAP or custom physics engine (match video’s momentum/bounce via spring physics).
  • Audio: Howler.js or native Web Audio for sync.
  • Backend (optional): Node.js/Express or Firebase for content sync across devices.
  • Character Set: Full split-flap compatible charset (A-Z, 0-9, !?.,-:;’”/%$& etc.).
  • Testing: Side-by-side video comparison; must pass frame-by-frame visual and audio sync tests against the original 32s demo.

6. Out of Scope (for v1 Exact Recreation)

  • Physical hardware integration.
  • Multi-board syncing.
  • Advanced AI content generation.
  • Subscription features (per original post).

7. Success Criteria

  • A user watching the recreated app on their TV cannot distinguish it from the original video demo.
  • All visual, animation, and audio details match exactly.
  • Full content update cycle (quote → weather → stats) works seamlessly with the same mechanical charm.
